# Многопользовательская чат-система (C++ / Docker / SQLite)

Современное клиент-серверное приложение для обмена сообщениями в реальном времени, построенное на архитектуре TCP/IP с использованием Modern C++ (C++17). Проект демонстрирует навыки проектирования распределённых систем, контейнеризации, безопасного сетевого взаимодействия и организации отказоустойчивой серверной логики.

---

## 🛠 Общий стек технологий

* **Язык:** C++17 (Modern C++: RAII, умные указатели, многопоточность)
* **Сборка:** CMake (Modern Presets API)
* **Контейнеризация:** Docker & Docker Compose (Multi-stage builds, Alpine Linux)
* **Сетевой стек (POSIX):**
    * `sys/socket.h` — низкоуровневое управление TCP-соединениями.
    * `netdb.h` — современное разрешение имен (DNS-aware) через `getaddrinfo`.
* **Протокол:** JSON over TCP (`nlohmann/json`)
* **Логирование:** `spdlog` (асинхронное потокобезопасное логирование)
* **База данных:** SQLite3 (персистентное хранение истории сообщений)

## Серверная часть (Backend)

Многопоточный TCP-сервер, выступающий в роли брокера сообщений и подсистемы хранения истории.

### Архитектурные особенности

#### Многопоточность

* Изоляция каждого клиента в отдельном `std::thread`
* Потокобезопасное управление списком клиентских дескрипторов через `std::mutex` и `std::lock_guard`

#### Постоянное хранение истории

* Автоматическое сохранение сообщений в SQLite
* При подключении нового клиента сервер загружает последние 10 сообщений (`ORDER BY id DESC`), обеспечивая сохранение контекста диалога
#### Безопасность
* **Санитизация ввода:** замена ESC-последовательностей (`\033`) на безопасные печатные символы для защиты серверного терминала
* **Защита от SQL-инъекций:** использование prepared statements (`sqlite3_prepare_v2`) и привязки параметров
* **Корректное завершение:** перехват сигнала `SIGINT (Ctrl+C)` с уведомлением клиентов, сохранением состояния и корректным освобождением ресурсов через RAII

## Клиентская часть (Frontend CLI)

Интерактивный консольный клиент с поддержкой асинхронного взаимодействия и интеллектуальной отрисовкой интерфейса.

### Ключевые возможности

#### Неблокирующий интерфейс и UX
* **Async Reception:** Разделение пользовательского ввода (`std::getline`) и приёма сетевых данных в независимые потоки. Это позволяет мгновенно отображать входящие сообщения, не прерывая процесс набора текста.
* **Smart Terminal UX:** Использование ANSI-последовательностей (`\033[2K\r`) для динамической очистки строки. При получении сообщения клиент аккуратно перерисовывает приглашение к вводу, предотвращая "разрыв" текста пользователя.

#### Сетевая гибкость
* **DNS-aware Connection:** Благодаря системному API `getaddrinfo`, клиент поддерживает подключение как по IP, так и по именам хостов. Это критически важно для работы в Docker, где связь между контейнерами идет по именам сервисов.
* **External Config:** Все параметры (хост, порт, логирование) вынесены в `config.json`, что позволяет менять среду окружения без перекомпиляции кода.

#### Надежность (Modern C++)
* **Rule of Five:** Полноценная реализация «Правила пяти» для класса клиента. Это гарантирует корректное владение сетевым дескриптором и исключает утечки ресурсов или двойное закрытие сокета при копировании/перемещении объектов.
* **RAII Management:** Автоматическое закрытие соединений и освобождение памяти при выходе объекта из области видимости.

## Развёртывание и запуск (Docker)

Проект полностью контейнеризирован. Финальные production-образы на базе Alpine Linux имеют размер менее 20 MB.

### Сборка через Docker Compose

Предпочтительный способ запуска полной инфраструктуры (сервер и клиент):

```bash
# Из корня проекта
docker compose up --build
```
**Примечание:** Чтобы взаимодействовать с клиентом, используйте команду `docker attach chat_client_container` в новом терминале.

### Ручной запуск (режим отладки)

Для тестирования в отдельных терминалах используется выделенная Docker-сеть:

```bash
# 1. Создание сети
docker network create chat_net

# 2. Запуск сервера (Терминал 1)
docker build -t my_server ./server
docker run -it --rm --network chat_net --name chat_server -p 9000:9000 my_server

# 3. Запуск клиента (Терминал 2)
docker build -t my_client ./client
docker run -it --rm --network chat_net --name client_1 my_client
```

**Примечание:** В `client/config.json` адрес сервера должен быть указан как `"chat_server"`.


### Локальная сборка через CMake (без Docker)

Если проект запускается в локальной Linux-среде или любой совместимой POSIX-системе, доступен прямой запуск без контейнеризации:

```bash
cmake --preset default
cmake --build --preset default
```

Данный способ рекомендуется для локальной отладки, профилирования, использования sanitizers и работы в IDE с поддержкой CMake Presets.

## Сценарий обмена сообщениями

```text
1. Клиент запускается и считывает config.json
2. Устанавливается TCP-соединение
3. На сервер отправляется AUTH-сообщение
4. Сервер восстанавливает последние 10 сообщений из SQLite
5. Начинается двусторонняя чат-сессия в реальном времени
6. При завершении сервер рассылает событие отключения
```
